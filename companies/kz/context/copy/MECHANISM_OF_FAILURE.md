# Mechanism of Failure — kz

> **Framework:** StoryBrand (Donald Miller) — toda história precisa de um vilão.
> Este documento nomeia o vilão da categoria: por que outras soluções falham.
> O herói (Mariana) só é salva pelo guia (kz) que entende como derrotar este vilão específico.

## O Vilão — em uma frase

> "A suposição de que consultório se resolve empilhando ferramentas — quando o que a sessão precisa é uma tela só."

**Por que esta é a formulação certa:**
- Ataca a **abordagem** (empilhamento), não o concorrente nominal — isso posiciona; atacar marcas viraria FUD fraco
- Contra-intuitiva: em uma categoria que vende "completude" e "tudo em um só lugar", a kz diz que o problema *é* a lógica aditiva
- Conecta literal com o Unique Mechanism (Sessão Inteira = tela só)

## As 3 formas de falhar

### 1. Fragmentação por Soma

- **O que acontece:** A solução (seja DIY — Google Agenda + WhatsApp + Meet + caderno — seja vertical — prontuário + agenda + videochamada + portal em módulos paralelos) é uma **coleção de telas** em vez de uma **experiência convergente**. A terapeuta opera 4-5 interfaces para fazer uma sessão.
- **Sintoma que o cliente vê:** *"Tá tudo espalhado. Perco tempo entre WhatsApp, Google Agenda, Google Meet e caderno — e no final do dia eu tô cansada de gerenciar, não de atender."* (AUDIENCE > Dor Camada 1)
- **Por que é inevitável nessa abordagem:** Produtos que crescem por mercado-adjacente empilham módulos para cobrir múltiplos perfis (clínica + solo + equipe). Cada módulo foi otimizado isoladamente. Nenhum foi desenhado com o *ato de atender* no centro — logo, converger todos numa tela não é possível como patch; só como arquitetura original.

### 2. Desenhado para o Gestor, Não para o Terapeuta

- **O que acontece:** A maior parte dos sistemas verticais foi originalmente concebida para clínicas com secretária/administrador. Interfaces pesadas de relatórios financeiros, NFS-e, BI, conta digital, dashboards — tarefas secretariais que deslocam a atenção. O terapeuta solo recebe isso como "bônus", mas é carga que rouba foco.
- **Sintoma que o cliente vê:** *"Queria só atender."* (AUDIENCE > Fala como) + *"Não consigo desligar do trabalho."* (AUDIENCE > Fala como)
- **Por que é inevitável nessa abordagem:** O mercado vertical líder nasceu servindo clínicas maiores (Doctoralia, iClinic, Amplimed) ou foi construído por engenheiros com consultoras psicólogas como input secundário (Clínica Ágil, PsicoManager). UX herda a centralidade do gestor. Nem o redesenho adianta — quando o framework mental do produto é "operação clínica", a sessão vira uma subtela.

### 3. Sistema Pesado Demais para Uso Real

- **O que acontece:** Verticais vendem "completude" para justificar preço. Resultado: terapeuta solo abre o sistema, encontra 30 telas, tenta usar 10, usa 3. Ou desiste antes no trial, ou paga e larga no mês 2. A promessa de "tudo em um lugar" se cumpre tecnicamente mas falha experiencialmente.
- **Sintoma que o cliente vê:** *"E se eu pagar e não usar? Vou cair em mais um sistema pesado que largo no mês 2?"* + *"Vale R$ 100+/mês sendo que uso Google Agenda de graça?"* (AUDIENCE > Objeções principais)
- **Por que é inevitável nessa abordagem:** Modelo de negócio dos verticais maduros pressiona para empilhar features — cada nova função é uma justificativa de pricing tier. Simplicidade deixa dinheiro na mesa. Resultado: quem oferece "completo" não consegue oferecer "suficiente".

## Como a Sessão Inteira derrota o vilão

A Sessão Inteira é a resposta arquitetural específica a cada uma das três falhas.

- **Contra Fragmentação por Soma:** a Sessão Inteira converge os contextos numa tela. Não é "mais um app na sua stack" nem "um sistema que integra com os outros". É o que **substitui** a stack de 5 apps. O atrito de alternar desaparece porque não há o que alternar.
- **Contra Desenhado para o Gestor:** a tela é pensada do ponto de vista da sessão — o que o terapeuta precisa *enquanto* atende (anotações atuais, notas passadas, planejamento, info do paciente). Solo não é caso secundário; é o caso único do produto. Funções secretariais pesadas (NFS-e, BI, conta digital) não existem ou ficam em segundo plano radical.
- **Contra Sistema Pesado:** a Sessão Inteira é *o que basta*. Simplicidade como disciplina — valor literal no `COMPANY_PROFILE.md`: *"se uma feature complica, não entra"*. A kz não empilha para justificar preço; simplifica para justificar uso.

## Narrativa de storytelling

Bloco plug-and-play para sales page / VSL / abertura de e-mail #2 da sequência.

```
Antes:     Mariana vivia entre WhatsApp, Google Agenda, Meet e caderno. No
           fim do dia, cansada de gerenciar — não de atender.

Descobriu: o problema não era ela ser desorganizada. Era a suposição de que
           consultório se resolve empilhando ferramentas — quando o que a
           sessão precisa é uma tela só.

Por que:   a categoria nasceu servindo clínicas e foi adaptada para solo.
           Os sistemas são coleções de módulos paralelos; a experiência é
           gestão, não atendimento.

Solução:   a kz construiu a Sessão Inteira — uma tela de atendimento onde
           todos os contextos já estão abertos e à mão. Não é um sistema
           mais leve. É um sistema desenhado com o ato de atender no centro.

Agora:     Mariana chega inteira na sessão. O paciente à frente, tudo que
           ela sabe dele ao alcance. Volta a ser só terapeuta.
```

**Ponto crítico da narrativa:** o "Descobriu" realoca a culpa. A persona interna está se culpando ("eu sou desorganizada / eu sou amadora"). A narrativa redireciona: não é você, é a categoria. Isso libera a compra — ela para de se culpar e começa a ver o sistema como aliado, não como mais uma prova de que ela "não dá conta".

- **Sales page:** seção "Por que você ainda está sofrendo" ANTES de apresentar a solução — bloco Antes + Descobriu + Por que
- **Email #2 da sequência de nutrição:** "O motivo real pelo qual seu consultório parece bagunçado (e não é culpa sua)"
- **Ads MOFU no Instagram:** reel/carrossel abrindo com "Se você acha que é desorganizada, leia isso" — aterrissa na narrativa de realocação
- **Post LinkedIn TOFU:** tema "o setor inteiro nos diz para empilhar ferramentas — e aqui está o que isso custa"
- **Call / conversa com terapeuta:** roteiro para discovery — investigar se a pessoa sente "sou desorganizada" (padrão fortíssimo) e espelhar com o vilão nomeado
- **Bastidores do founder (Instagram / blog):** post em 1ª pessoa do fundador sobre por que construiu a kz — âncora na Fragmentação por Soma como experiência vivida

## Regras de uso

1. **Nunca nomear concorrentes como vilões.** O vilão é a abordagem, nunca Corpora / Clínica Ágil / iClinic. Atacar marca vira FUD; atacar padrão é posicionamento.
2. **Usar o bloco Descobriu como gatilho emocional.** A realocação de culpa (*"não é você, é a categoria"*) é o momento de maior valor emocional na copy — merece espaço e pausa.
3. **Citar o Mechanism por nome** ("Sessão Inteira") quando o texto oferecer a solução — reforço da marca proprietária.
4. **Tom acolhedor, mesmo no diagnóstico.** BRAND_VOICE > Regra 1: *"Respire antes de vender."* Nomear o vilão não é atacar a Mariana — é dar nome ao que ela já sentia.
