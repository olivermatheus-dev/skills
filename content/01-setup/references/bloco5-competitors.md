# Bloco 5 — Mapeamento de Concorrentes

## Objetivo

Mapear concorrentes diretos e indiretos com profundidade suficiente para diferenciar o conteúdo. Este bloco combina inputs do usuário com pesquisa ativa na web.

## Sequência de Perguntas

| # | Pergunta | Ajuda |
|---|---|---|
| 5.1 | Quais concorrentes você conhece? (diretos) | "Empresas que seus clientes mencionam, ou que aparecem quando pesquisa seu setor" |
| 5.2 | Tem empresas que não são concorrentes diretas, mas resolvem o mesmo problema de outro jeito? | Ex: se vende CRM, planilhas e WhatsApp desorganizado são concorrentes indiretos |
| 5.3 | Quer que eu pesquise mais concorrentes? | **Pesquisa ativa na web** |
| 5.4 | Para cada concorrente: O que acha que fazem bem? | "Pode ser 'o site deles é bonito' ou 'têm muito conteúdo no Instagram'" |
| 5.5 | Para cada concorrente: O que fazem mal ou não fazem? | "Esses gaps viram oportunidades de conteúdo" |
| 5.6 | Quer que eu analise as redes sociais dos concorrentes? | **Pesquisa ativa de conteúdo** |
| 5.7 | Tem alguma empresa (de qualquer setor) cujo conteúdo você admira? | "Não precisa ser concorrente — pode ser referência de estilo" |

## Protocolo de Pesquisa Ativa

### Pesquisa de concorrentes (quando aceitar 5.3):

1. Buscar na web usando combinações:
   - "[categoria do produto] alternativas Brasil"
   - "[setor] software/ferramenta Brasil"
   - "melhor [categoria] para [público-alvo]"
   - "[produto do usuário] vs" (autocomplete revela concorrentes)
2. Para cada empresa encontrada: coletar nome, site, descrição em 1 linha, posicionamento aparente
3. Apresentar lista: "Encontrei estas empresas. Quais você considera concorrentes?"
4. Confirmar antes de incluir no mapa

### Análise de conteúdo dos concorrentes (quando aceitar 5.6):

1. Buscar: "[nome concorrente] LinkedIn" e "[nome concorrente] Instagram"
2. Analisar:
   - Tipo de conteúdo predominante (educativo, produto, cases, memes, bastidores?)
   - Frequência estimada (diário, 3x semana, semanal?)
   - Formatos usados (carrossel, vídeo, texto longo, stories?)
   - Tom de voz percebido (formal? casual? técnico?)
   - Engajamento visível (muitos comentários? poucos?)
3. Sintetizar para o usuário — nunca reproduzir conteúdo alheio
4. Identificar gaps: "Nenhum deles faz [X]. Isso é uma oportunidade."

## Consolidação de Gaps

Ao final, cruzar TODOS os concorrentes e identificar:
- Temas que ninguém cobre (ou cobre superficialmente)
- Formatos que ninguém usa no setor
- Ângulos inexplorados (contrarian, dados, storytelling)
- Frequências e horários mal atendidos

Esses gaps alimentam diretamente o Bloco 6 (Pilares de Conteúdo).
