# Bloco 6 — Pilares de Conteúdo

## Objetivo

Definir os pilares temáticos e a estratégia de conteúdo. Este é o bloco mais assistido — a skill deve SUGERIR pilares baseados em tudo que já foi preenchido.

## Sequência de Perguntas

| # | Pergunta | Ajuda |
|---|---|---|
| 6.1 | **Sugestão automática:** "Baseado em tudo que conversamos, sugiro estes pilares. O que acha?" | Cruzar PRODUCT × AUDIENCE × COMPETITOR para gerar 4-6 pilares |
| 6.2 | Para cada pilar: Quer manter, ajustar ou remover? | Explicar cada pilar com justificativa |
| 6.3 | Quer adicionar algum pilar que não sugeri? | "Pode ser bastidores, cultura, ou um tema que você domina" |
| 6.4 | Com que frequência pretende postar por semana? | Sugerir: "3-5 posts/semana em 1-2 plataformas é um bom início" |
| 6.5 | Qual proporção educativo vs. produto vs. bastidores? | Sugerir: "50% TOFU / 30% MOFU / 20% BOFU" com explicação simples |

## Lógica de Sugestão Automática (6.1)

O coração deste bloco é a sugestão inteligente:

```
Para cada feature principal do PRODUCT_CONTEXT:
  → Cruzar com as dores do AUDIENCE_PROFILES que essa feature resolve
  → Verificar se algum concorrente no COMPETITOR_MAP já domina esse tema
  → Se gap encontrado: prioridade alta
  → Se concorrente domina: prioridade menor (mas pode entrar com ângulo diferente)

Gerar 4-6 pilares no formato:
  Pilar: [Nome]
  Cobre: [subtemas]
  Dor que endereça: [dor do audience profiles]
  Persona principal: [qual]
  Gap de mercado: [sim/não, baseado no competitor map]
  Prioridade sugerida: [alta/média]
```

### Como apresentar

Para cada pilar sugerido, explicar em 2-3 frases com justificativa clara:

"**Pilar: Organização Comercial** — Este pilar cobre funil de vendas, processos e follow-up. Sugiro porque sua persona principal ('O Gestor Sobrecarregado') tem como dor principal a falta de processo, e nenhum dos 3 concorrentes mapeados publica conteúdo aprofundado sobre isso. Prioridade: alta."

### Pilares comuns por tipo de negócio

Se faltar dados para cruzamento, sugerir baseado no tipo de negócio:

**SaaS B2B:**
1. Problema/dor (educação sobre o problema que resolve)
2. Produto/solução (features, tutoriais, demos)
3. Bastidores/cultura (transparência, jornada do fundador)
4. Setor/mercado (tendências, dados, análises)

**Serviços/Consultoria:**
1. Metodologia (frameworks, processos, como funciona)
2. Cases/resultados (provas sociais, antes/depois)
3. Educação do mercado (mitos, erros comuns)
4. Autoridade (opinião, posicionamento, tendências)

**E-commerce:**
1. Produto/curadoria (novidades, combinações, usos)
2. Lifestyle (como o produto se encaixa na vida)
3. Bastidores (fabricação, equipe, valores)
4. Comunidade (UGC, depoimentos, interação)

## Exercício de Proporção (6.5)

Explicar TOFU/MOFU/BOFU de forma acessível:

- **TOFU (Topo — consciência):** "Conteúdo para quem NÃO sabe que tem um problema. O objetivo é fazer pensar."
- **MOFU (Meio — consideração):** "Conteúdo para quem SABE que tem um problema e busca solução."
- **BOFU (Fundo — decisão):** "Conteúdo para quem está DECIDINDO entre opções."

Sugestões de mix por estágio do negócio:
- **Lançamento:** 30% TOFU / 30% MOFU / 40% BOFU (priorizar conversão)
- **Crescimento:** 50% TOFU / 30% MOFU / 20% BOFU (priorizar alcance)
- **Estável:** 40% TOFU / 40% MOFU / 20% BOFU (equilibrar)
